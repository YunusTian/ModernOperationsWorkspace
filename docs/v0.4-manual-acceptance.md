## 0. 准备

```powershell
# Windows PowerShell
$env:OPENAI_API_KEY = "sk-真key"
# CLI 与 Desktop 都会读同一个环境变量（除非配置 api_key_env 覆盖）
```

`~/.mow/config.json`（或 `MOW_CONFIG`）建议内容：

```json
{
  "plugins": {
    "ai": {
      "enabled": true,
      "settings": {
        "providers": [
          {
            "name": "openai",
            "kind": "openai",
            "options": {
              "base_url": "https://api.openai.com/v1",
              "api_key_env": "OPENAI_API_KEY",
              "default_model": "gpt-4o-mini",
              "models": ["gpt-4o-mini"]
            }
          }
        ]
      }
    }
  },
  "ai": {
    "allowed_tools": ["probe.info"]
  }
}
```

> `allowed_tools` 里放什么由验收人决定；上例是搭配 [tests/e2e/ai_e2e_test.go](../tests/e2e/ai_e2e_test.go) 的 `probePlugin`。若无 in-process 插件，可留空进入纯对话模式，跳过场景 4。

## 场景 1 — 一次性对话主路径

- [ ] `mow ai providers` 输出应包含一行 `openai\tchat=true stream=true tools=true ...`
- [ ] `mow ai ask "用一句话介绍 MOW"`
  - 期望：终端打印一段合理回答，退出码 0
  - 期望：`~/.mow/logs/` 或 stderr 里有一条 `ai.loop.start` + 一条 `ai.round.end`（含 `usage_tokens`） + 一条 `ai.loop.end`（`finish_reason=stop`）
  - 记录 total_tokens：`________`
- [ ] Desktop 打开 AI 页面：
  - 顶部下拉里选 `openai`
  - 点击 **Ask** 按钮 → 出现 `rounds:1 tokens:… stop` 徽章
  - 点击 **Send** 按钮 → 消息以打字机效果展现

## 场景 2 — 用户取消 / Ctrl+C

- [ ] 打开一次交互式 chat：
  ```
  mow ai chat
  > 请用 500 字讲讲 SSH 协议历史
  ```
  在模型开始返回后立即按 **Ctrl+C**：
  - 期望：进程 5 秒内退出，不吞出错误
  - 期望：日志出现 `ai.loop.end finish_reason=canceled`（或 SDK 层 `ERROR`）
- [ ] Desktop：Send 期间点击 **Stop**
  - 期望：会话结束，`Stop` 按钮消失，Send/Ask 按钮回归

## 场景 3 — 限流 / 网络故障

前置：把 `MOW_HTTP_CLIENT_TIMEOUT` / 或短时间内触发 OpenAI 限流（连续 20 次 ask）。

- [ ] 观察 stderr / logs：应出现两次 `warn: openai retry attempt=… code=RATE_LIMITED backoff=…ms`，然后要么成功，要么最终报 `AI_RATE_LIMITED`
- [ ] Desktop：Ask 后错误红条显示 `AI_RATE_LIMITED` 或 `AI_UPSTREAM_ERROR` 前缀
- [ ] Desktop：错误红条右侧的 **Retry** 按钮点击后能重试，且重试成功后徽章会更新

## 场景 4 — 只读工具调用（tool-use 闭环）

前置：`allowed_tools` 至少一项 Read Command（例如 `probe.info` 或 `ssh.connections.list`），或临时用一份 in-process 插件。

- [ ] `mow ai ask --json "call probe.info with key=platform"`
  - 期望：JSON 里包含 `usage`、`finish_reason=stop`
  - 日志顺序（`grep component=ai.audit`）：
    ```
    ai.loop.start   tools=[probe.info]
    ai.round.start  round=1
    ai.round.end    finish_reason=tool_calls tool_calls=1
    ai.tool.call    tool_name=probe.info duration_ms=… args_digest={"key":"platform"} audit_id=…
    ai.round.start  round=2
    ai.round.end    finish_reason=stop tokens=…
    ai.loop.end     finish_reason=stop
    ```
- [ ] Desktop Ask 相同问题：
  - 徽章显示 `rounds:2 tools:1 tokens:… stop`
  - "Tool call" 折叠区可展开看到工具调用参数
- [ ] 尝试把 Dangerous 命令加入 `allowed_tools`（例：`docker.rm`）后重启 Desktop：
  - 期望：AI 页面顶部出现红色横幅 `AI tool-use disabled` 与错误码 `AI_TOOL_NOT_READABLE`
  - Ask / Send 应仍能工作（只是没有工具能被调用）

## 1. `mow plugin dev [--watch]` 热重载命令

- [x] 新增 [apps/cli/plugin_watch.go](../apps/cli/plugin_watch.go)：`stageDevPackage` 复用 §6 的 build + `platforms[]` 裁剪 + checksum 注入
- [x] 数据流：`stage → core/plugin.Lifecycle.Install`（首次）或 `Update`（后续，原子替换 + 失败自动回退）到 `<PluginsDir>/<id>` → `SetEnabled(true)`
- [x] `--watch` 模式：轮询 `<srcDir>` 下的 `*.go / *.yaml / *.yml / *.json / go.mod / go.sum`；跳过 `.git / vendor / dist / node_modules`
- [x] `--interval` 参数控制轮询间隔（默认 500ms）
- [x] 强制 host GOOS/GOARCH：Manifest 的 `platforms[]` 会裁剪成当前平台条目；跨平台 dev 参数校验阶段直接拒绝
- [x] build 失败**不退出 watch loop**：stderr 打印错误但循环继续，允许改回来继续迭代
- [x] 依赖注入 `packageBuilder`：生产实现 `goBuilder{}` 走 `runGoBuild`；测试实现只写占位字节，单测因此无需 `go` 工具链
- [x] 在 [apps/cli/plugin.go](../apps/cli/plugin.go) 的 `newPluginCmd` 注册 `newPluginDevCmd(holder)`，与 `init / lint / package / validate / list / install / update / uninstall / enable / disable / doctor / search / catalog / config` 并列

## 9. 版本策略与非目标

- 本版本是 **v0.5 的最后一个 DX 增量**，不引入新特性：SDK / Manifest / Plugin Protocol / Schema / SecretStore / Catalog 契约全部冻结不动
- [VERSION](../VERSION) 由 `0.5.3` 上调为 `0.5.4`；[sdk/version/version.go](../sdk/version/version.go) 同步
- 官方四款插件（ssh / docker / ai / pve）`plugin.json.version` 同步为 `0.5.4`；`compatibility.core` 与 `compatibility.sdk` 兼容范围保持不变（`>=0.4.0,<0.6.0` 与 `>=0.5.0,<0.6.0`）
- 兼容性：已发布的 v0.5.3 catalog 与二进制无需重打；新 tag 只是补齐 DX 命令与文档

## 10. 非目标与延后项

- ⏸ `plugin dev` 引入 `fsnotify` 原生 file watcher（当前轮询对本地开发中小型仓库完全够用）→ 若未来社区反馈需求强烈，v0.7+ 再评估
- ⏸ `plugin package` 单测（需真实 `go build`，与 `plugin dev` 用假 builder 规避的思路不同）→ 保持依赖 Release Smoke 兜底
- ⏸ 复杂 PVE 创建向导 / 存储迁移 / Dangerous 删除 → v0.7
- ⏸ Workflow 2.0（版本化 / 子工作流 / 审批 / 调度 / SQLite 历史）→ v0.6

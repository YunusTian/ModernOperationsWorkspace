# v0.5.0 验收清单 — Plugin Manifest 与包格式

- 状态：Planned（骨架，待实现）
- 目标版本：v0.5.0
- 更新日期：2026-07-11
- 范围：只做 Manifest schema、`mow plugin validate` 命令、运行时对 Manifest 的强校验。**不做**下载 / 安装 / Catalog / 配置 UI（这些属于 v0.5.1 / v0.5.2）。
- 相关文档：[开发计划 §4.2](./development-plan-v0.5-v1.0.md#42-v050plugin-manifest-与包格式) · [Plugin System RFC](./plugin-system.md)

## 1. Manifest 规范与 Schema

- [ ] `sdk/manifest/plugin.schema.json` 作为 Manifest 的权威 JSON Schema（draft 2020-12）
- [ ] Schema 覆盖字段：`id` / `name` / `version` / `author` / `license` / `homepage` / `description`
- [ ] Schema 覆盖兼容范围：`compatibility.core` / `compatibility.sdk` / `compatibility.protocol`（语义化 range 字符串）
- [ ] Schema 覆盖平台矩阵：`platforms[]`，每项含 `os` / `arch` / `entrypoint` / `checksum`
- [ ] Schema 覆盖能力摘要：`commands[]`（id / permission）、`connectionTypes[]`、`permissions[]`
- [ ] Schema 覆盖 `settingsSchema`（内嵌 JSON Schema，v0.5.2 才驱动 UI，v0.5.0 只校验其合法性）
- [ ] Schema 覆盖资源清单：`recipes[]` / `workflows[]`（相对包内路径）
- [ ] Schema 覆盖数据版本与迁移入口：`dataVersion` / `migrations[]`
- [ ] Schema 覆盖发布来源与签名：`source.url` / `source.tag` / `signature.algorithm` / `signature.value`（v0.5.0 允许缺省）
- [ ] Schema 有单元测试，覆盖合法 / 非法样本，错误码稳定

## 2. Manifest 加载与解析

- [ ] `sdk/manifest` 提供 `Load(path) (Manifest, error)` / `Parse(data) (Manifest, error)`
- [ ] 解析失败返回 `PLUGIN_MANIFEST_INVALID` + 定位（字段 / offset）
- [ ] 兼容范围解析基于 semver constraint（如 `>=0.5.0,<0.6.0`）
- [ ] Manifest 与 gRPC Metadata 的字段映射有单向 diff 工具
- [ ] SDK 契约测试覆盖 Manifest 反序列化、兼容范围解析、错误码稳定性
- [ ] 未新增破坏性 SDK 或 Plugin Protocol 变更（Manifest 仅补充，运行时协议不变）

## 3. `mow plugin validate` 命令

- [ ] `mow plugin validate <path>` 支持传入 `plugin.json` 或包目录
- [ ] 输出人类可读报告 + `--json` 结构化输出
- [ ] 稳定错误码：`PLUGIN_MANIFEST_INVALID` / `PLUGIN_CHECKSUM_MISMATCH` / `PLUGIN_ENTRYPOINT_MISSING`
- [ ] 校验 checksum：对每个 `platforms[].entrypoint` 计算 SHA-256 并与 Manifest 声明比对
- [ ] 校验 `recipes[]` / `workflows[]` 引用的相对路径实际存在
- [ ] `-v` verbose 模式打印所有通过项，便于插件开发者调试
- [ ] 单元测试覆盖 happy path + 五类失败分支

## 4. 运行时强校验

- [ ] Plugin Manager 在 Load 阶段读取包目录的 `plugin.json`
- [ ] Manifest.ID / Version 与 gRPC `sdk.Metadata` 返回值不一致 → 拒绝启用，错误码 `PLUGIN_MANIFEST_MISMATCH`
- [ ] `compatibility.core` 不满足当前 Core 版本 → 拒绝加载，错误码 `PLUGIN_INCOMPATIBLE`
- [ ] `compatibility.sdk` / `compatibility.protocol` 不满足 → 同错误码
- [ ] 拒绝发生在启动子进程**之前**（不允许先起进程再拒绝）
- [ ] 拒绝原因附带具体字段与期望值，便于诊断
- [ ] 单元测试覆盖 Mismatch / Incompatible / Happy 三条主路径

## 5. 官方插件补齐 plugin.json

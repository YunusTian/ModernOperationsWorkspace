## 1. 交付摘要

v0.2 的核心目标是把 **Workflow Engine** 从设计稿落地为 MVP，覆盖：

- 静态数据模型 + `Validate`
- YAML DSL 严格模式解析（`KnownFields(true)`）
- `${...}` 变量插值（inputs / steps.out）
- Runner 主循环 + 三阶段 OnStep 回调
- CLI 子命令 `mow workflow validate|run`（含彩色进度）
- Desktop `WorkflowPage`（拖拽 → 表单 → 实时日志流）
- E2E：`examples/workflows/deploy-static-site.yaml` 通过 fake SSH server 走通

## 4. 手动验收（20/20）

### 4.1 CLI

- [x] `mow workflow validate examples/workflows/deploy-static-site.yaml` 输出 `ok: deploy.static-site (3 steps)`
- [x] 未知字段 YAML（如 `retry:`）→ 严格模式报错，路径含字段名
- [x] `mow workflow run --input k=v` 顺序执行三步，`▶/✓` 彩色打印，退出码 0
- [x] 缺失 `--target` → Workflow 引擎按用户输入拒绝或降级
- [x] 缺失 required input → CLI 层直接失败：`missing required input: <name>`
- [x] `--json` 抑制进度、仅输出最终 Result
- [x] `--no-color` / `NO_COLOR=` 关闭 ANSI

### 4.2 Desktop

- [x] Workflow tab 常驻，无 target 也可打开
- [x] 拖拽 YAML → 立即触发 `WorkflowValidate` 并展示 id / name / description / step_count
- [x] `inputs` 表单：`type` 显示、`required *` 标记、`default` 占位符
- [x] Run 后 `workflow:<sess>:step` 事件推送 `▶/✓/✗`
- [x] 错误 step 展示 `[ERROR_CODE]` + 消息，`workflow:<sess>:done` 收尾

### 4.3 变量插值边界

- [x] `${inputs.port + 1}` 计算成功
- [x] `${steps.upload.out.bytes_sent}` 保留原始类型（int）
- [x] `${inputs.missing}` → `InterpolationError` 带 offset
- [x] `bad ${inputs.` → unterminated 报错
- [x] 深度嵌套 map + slice 内的 `${}` 均被替换

### 4.4 E2E fake SSH server

- [x] `TestWorkflow_DeployStaticSite_EndToEnd` 3 步顺序执行
- [x] fake server 收到的每条 cmdline 均已插值（无残留 `${`）
- [x] `OnStep` 6 次回调，Start 顺序 `backup → upload → health`

## 5. 未纳入 v0.2（已在 [workflow.md §7.5](./workflow.md#75-尚未实现-v03) 明确 v0.3+）

- `parallel: true` — 组内并行
- `when: <expr>` — 条件分支
- `on_failure` / `onFailure` — 补偿动作
- `retry: { max, backoff }` — 重试
- `notify: { channel, target }` — 通知
- `rollback: [step_ids]` — 声明式回滚
- 单 step 独立 `target`
- 执行历史 SQLite 持久化
- Workflow 版本化 / 迁移（v0.4+）

## 1. CLI 八条命令齐全

- [x] `mow plugin list` —— 表格 + `--json`，含 `installed_at`、`enabled` 状态（[apps/cli/plugin.go](../apps/cli/plugin.go)）
- [x] `mow plugin search [query] [--all] [--refresh] [--os] [--arch] [--json]` —— 多源 catalog + 平台/兼容性过滤（[apps/cli/plugin_catalog.go](../apps/cli/plugin_catalog.go)）
- [x] `mow plugin install <package|id[@version]>` —— 本地路径 + Catalog 两种输入，`--path` / `--catalog` 强制模式（[apps/cli/plugin.go](../apps/cli/plugin.go)）
- [x] `mow plugin update <package|id[@version]>` —— 同上，走 `Lifecycle.Update` 原子替换 + 备份回退
- [x] `mow plugin enable <id>` / `mow plugin disable <id>` —— 只改 `.state/<id>.json`，冷启动禁用生效
- [x] `mow plugin uninstall <id> [--purge]` —— 默认保留 `.state`，`--purge` 才清 state 与 `.bak`（[core/plugin/lifecycle.go](../core/plugin/lifecycle.go)）
- [x] `mow plugin doctor` —— 单插件深度诊断，结构化错误码 + `Details`
- [x] `mow plugin catalog list|refresh` —— 展示已配置源、拉取（[apps/cli/plugin_catalog.go](../apps/cli/plugin_catalog.go)）
- [x] 稳定错误码：`PLUGIN_MANIFEST_INVALID` / `PLUGIN_INCOMPATIBLE` / `PLUGIN_PACKAGE_INVALID` / `PLUGIN_CHECKSUM_MISMATCH` / `PLUGIN_UPDATE_ROLLBACK`

## 2. Catalog 客户端

- [x] `core/plugin/catalog.Catalog` schema：`catalogVersion` / `entries[].versions[].platforms[]`；`Parse` 拒绝未知字段 / BOM / 重复 id（[core/plugin/catalog/catalog.go](../core/plugin/catalog/catalog.go)）
- [x] 过滤：按 `runtime.GOOS/GOARCH` + `compatibility.core` semver 约束 + yanked；`LatestFor` 按 semver 降序
- [x] `catalog.Client` 支持 `http/https/file` 三种 scheme、多源合并、单源失败不阻塞
- [x] 离线缓存：sha256(name+URL) 命名 + 原子 rename；网络失败 `force=false` 时回退缓存，`force=true` 时报错但保留缓存
- [x] `MaxBytes` 防御性截断（默认 256 MiB）
- [x] `catalog_test.go` 覆盖 Parse happy/未知字段/重复 id、过滤器、`LatestFor`、HTTP + 缓存回退、坏 JSON 保留旧缓存、`file://` scheme

## 3. Download / Installer

- [x] `plugin.Download(ctx, url, checksum, opts)` 统一入口：HTTP/HTTPS/file → sha256 校验 → 解压 tar.gz / tar / zip / 裸 plugin.json（[core/plugin/download.go](../core/plugin/download.go)）
- [x] 强制 `sha256:<64hex>` 格式；hex 大小写规范化
- [x] 解压安全：`safeJoin` 拒 `..` 与绝对路径穿越；symlink/hardlink 一律拒绝
- [x] 支持归档顶层多一层目录（如 `sample-plugin-0.5.1/`）
- [x] `Installer.Install(ctx, "id[@version]")` / `Update(...)` 串联 Catalog + Download + Lifecycle（[core/plugin/installer.go](../core/plugin/installer.go)）
- [x] 未指定 version → 走 `LatestFor`（含平台/兼容性）；指定 version → 精确匹配且仍走过滤器
- [x] `LooksLikeCatalogRef(arg)` 用于 CLI 自动路由本地路径 vs catalog

## 4. Lifecycle：Install / Update / Uninstall

- [x] Install：临时目录 → `ValidatePackage` → 原子 `rename` → `writeState`（保持 `enabled=false`）
- [x] Update：临时暂存 → 备份原目录到 `.update-backup-<id>-*` → 原子替换 → 写 state；任一步失败 `rollbackUpdate` 恢复备份并清理暂存
- [x] Uninstall：ID 校验（防路径穿越）→ 存在性检查 → `rename` 到临时目录 → `RemoveAll`；`--purge` 时清 `.state`
- [x] `enabled` 状态在 Update 后保留（Lifecycle 读旧 state 再合并到新 manifest）
- [x] `lifecycle_test.go`：install 幂等失败、update 成功替换、update 失败保留原版本、uninstall 保留/purge state、拒绝缺失/非法 id

## 8. 安全承诺

- [x] 下载后必须校验 SHA-256（catalog + `plugin.json` 双重）
- [x] 默认只允许可信来源：`Default()` 预置 `official` 且 `trusted=true`；用户显式添加的私有源默认 `trusted=false`
- [x] 卸载不静默删除数据：`.state/<id>.json` 保留，`--purge` 才清
- [x] 升级失败自动回退旧二进制 + 旧 Manifest；备份目录成功后才清理
- [x] `PLUGIN_CHECKSUM_MISMATCH` 拒绝写入插件目录，磁盘不留半成品（`plugin_install_e2e_test.go` 覆盖）

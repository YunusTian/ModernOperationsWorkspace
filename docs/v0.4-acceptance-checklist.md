# v0.4 验收清单 — AI 可用闭环

- 状态：In Progress
- 目标版本：v0.4.0
- 更新日期：2026-07-11
- 发布定义：至少一个真实 AI Provider 可通过 CLI 和 Desktop 使用，并能在宿主侧安全调用只读 MOW Command。

## 1. 已完成的基础

- [x] `sdk.Provider`、Chat、流式回调和 tool-call 值对象
- [x] `plugins/ai` 独立插件骨架
- [x] `mock` Provider
- [x] `ai.list_providers` / `ai.chat` / `ai.chat_stream`
- [x] AI 插件纳入 build、vet、test CI
- [x] AI 插件单元测试覆盖率达到 70% 以上

## 2. Provider（发布阻断）

- [x] 实现 OpenAI-compatible Provider
- [x] 支持 `base_url`、`api_key_env`、默认模型和自定义请求头
- [x] 支持一次性与流式 Chat
- [x] 支持 tool calls
- [x] 尊重 context 取消和请求超时
- [x] 将 401 / 403 / 429 / 5xx 映射为稳定错误码
- [ ] 仅对可重试错误执行有上限的退避重试
- [x] API Key 只从环境变量读取，且不进入请求错误与响应
- [x] 使用假 HTTP Server 完成契约测试，不依赖公网和真实密钥

## 3. Tool-use（发布阻断）

- [x] 确定由宿主 Core 编排 tool-use，见 [ADR-0001](./adr/0001-host-side-ai-tool-orchestration.md)
- [ ] AI Tool 定义由 CommandSpec 生成，不接受模型任意声明工具
- [x] 第一阶段只允许显式 allowlist 中的 Read Command
- [x] 下游请求使用 `CallerAI` 并继承父 audit 关联信息
- [x] 参数经过 Command Engine schema 校验
- [ ] 限制最大轮数、单轮调用数、输出大小和总超时
- [x] 禁止 AI 调用 `ai.*`，防止递归
- [x] Dangerous Command 不进入 v0.4 工具目录
- [ ] 完整记录模型决策、工具名称、参数摘要、结果与错误

## 4. CLI（发布阻断）

- [x] `mow ai providers`
- [ ] `mow ai chat`，支持多轮输入与流式输出
- [x] `mow ai ask <prompt>`，支持一次性对话
- [x] `ask` 支持 provider、model、timeout 参数
- [ ] Ctrl+C 能取消模型请求和正在执行的工具
- [ ] CLI 命令测试覆盖参数、输出、取消和错误路径

## 5. Desktop（发布阻断）

- [ ] AI 页面及导航入口
- [ ] Provider / Model 选择
- [ ] 流式消息与停止生成
- [ ] Tool call 状态和结果展示
- [ ] Dangerous 请求确认卡片或明确拒绝提示
- [ ] 错误、重试和 token usage 展示
- [ ] 至少覆盖后端绑定测试与前端关键状态测试

## 6. 安全、质量与发布（发布阻断）

- [ ] AI 配置 schema 可被 CLI/Desktop 共同消费
- [ ] 敏感字段递归脱敏
- [ ] Windows 与 Linux 完成 mock + fake-provider E2E
- [ ] `go test ./...`、race、vet、前端 build/test 全部通过
- [ ] README、CHANGELOG、roadmap、CLI、Desktop 版本状态一致
- [ ] 发布产物包含 AI 插件和校验文件
- [ ] 完成人工验收：真实 Provider 对话、取消、限流错误、只读工具调用

## 7. 明确不纳入 v0.4

- Anthropic 原生 Provider；可在 OpenAI-compatible 主链路稳定后追加
- MCP Server / Client
- RAG、Embedding、Vector Store
- 自动执行 Dangerous Command
- 多用户、远程集中式会话与团队权限
- PVE / Kubernetes / Database 新插件

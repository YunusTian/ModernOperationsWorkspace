# v0.4 验收清单 — AI 可用闭环

- 状态：Released（v0.4.0，2026-07-11；剧本见 [v0.4-manual-acceptance.md](./v0.4-manual-acceptance.md)）
- 目标版本：v0.4.0
- 更新日期：2026-07-11
- 发布定义：至少一个真实 AI Provider 可通过 CLI 和 Desktop 使用，并能在宿主侧安全调用只读 MOW Command。

## 1. 已完成的基础

- [x] `sdk.Provider`、Chat、流式回调和 tool-call 值对象
- [x] `plugins/ai` 独立插件骨架
- [x] `mock` Provider
- [x] `ai.list_providers` / `ai.chat` / `ai.chat_stream`
- [x] AI 插件纳入 build、vet、test CI
- [x] AI 插件单元测试覆盖率达到 70% 以上

## 2. Provider（发布阻断）

- [x] 实现 OpenAI-compatible Provider
- [x] 支持 `base_url`、`api_key_env`、默认模型和自定义请求头
- [x] 支持一次性与流式 Chat
- [x] 支持 tool calls
- [x] 尊重 context 取消和请求超时
- [x] 将 401 / 403 / 429 / 5xx 映射为稳定错误码
- [x] 仅对可重试错误执行有上限的退避重试（P0-4：`retryPolicy`，429/5xx 走 500ms→1s→cap=5s，401/403 直接失败）
- [x] API Key 只从环境变量读取，且不进入请求错误与响应
- [x] 使用假 HTTP Server 完成契约测试，不依赖公网和真实密钥

## 3. Tool-use（发布阻断）

- [x] 确定由宿主 Core 编排 tool-use，见 [ADR-0001](./adr/0001-host-side-ai-tool-orchestration.md)
- [x] AI Tool 定义由 CommandSpec 生成，不接受模型任意声明工具（P0-1：`buildToolSpec` 从 allowlist 派生并校验 Read/非流式/非 ai.*）
- [x] 第一阶段只允许显式 allowlist 中的 Read Command
- [x] 下游请求使用 `CallerAI` 并继承父 audit 关联信息
- [x] 参数经过 Command Engine schema 校验
- [x] 限制最大轮数、单轮调用数、输出大小和总超时（P0-2：`MaxRounds`/`MaxCallsPerRound`/`MaxTotalCalls`/`MaxResultBytes`/`Timeout` 五道护栏 + 稳定错误码）
- [x] 禁止 AI 调用 `ai.*`，防止递归
- [x] Dangerous Command 不进入 v0.4 工具目录
- [x] 完整记录模型决策、工具名称、参数摘要、结果与错误（`core/ai.Auditor` 5 类事件 + `SlogAuditor` + 参数摘要经 Redactor 脱敏）

## 4. CLI（发布阻断）

- [x] `mow ai providers`
- [x] `mow ai chat`，支持多轮输入与流式输出
- [x] `mow ai ask <prompt>`，支持一次性对话（v0.4：改走宿主 orchestrator，自带审计与脱敏）
- [x] `ask` 支持 provider、model、timeout 参数
- [x] Ctrl+C 能取消模型请求
- [x] CLI 命令测试继续覆盖真实插件错误路径（`apps/cli/ai_test.go`：Dangerous / Streaming / Recursive AI / Unknown Tool / ctx cancel / provider 错误码传播 共 7 用例）

## 5. Desktop（发布阻断）

- [x] AI 页面及导航入口
- [x] Provider / Model 选择
- [x] 流式消息与停止生成
- [x] Tool call 状态和结果展示
- [x] Dangerous 请求确认卡片或明确拒绝提示（`AIStatus` 后端接口 + AIPage 顶部 `ai-config-alert` 横幅，orchestrator 构造错误码直接展示）
- [x] 错误与 token usage 展示（`UsageBadges`: rounds / tools / tokens / finish_reason；`.ai-error` 展示编排错误码）
- [x] 新增 `AIAsk` 走宿主 orchestrator，前端 Ask 按钮完成 usage 展示
- [x] 重试交互（Retry 最后一条 user message，红条右侧 `.ai-retry` 按钮）
- [x] 覆盖后端绑定边界与前端流式关键状态测试

## 6. 安全、质量与发布（发布阻断）

- [x] AI 配置 schema 可被 CLI/Desktop 共同消费（`core/config.AIConfig`：allowed_tools + 各上限）
- [x] 敏感字段递归脱敏（P0-3：`command.RedactParams` 递归支持 object / array / 整树 sensitive；orchestrator 回写历史前脱敏）
- [x] Windows 与 Linux 完成 mock + fake-provider E2E（`tests/e2e/ai_e2e_test.go`：mock 主路径 / OpenAI happy path / 429 有上限退避重试 / tool_call 白名单命令 共 4 用例）
- [x] `go test ./...`、race、vet、前端 build/test 全部通过（当前主分支全绿；需在 CI 上锁定 v0.4 tag）
- [x] README、CHANGELOG、roadmap、CLI、Desktop 版本状态一致（v0.4.0 badge + [v0.4.0] CHANGELOG 段 + checklist 引用）
- [x] Release Workflow 已包含 AI 插件和通用校验文件
- [x] 完成人工验收：真实 Provider 对话、取消、限流错误、只读工具调用（剧本见 [v0.4-manual-acceptance.md](./v0.4-manual-acceptance.md)）

## 7. 明确不纳入 v0.4

- Anthropic 原生 Provider；可在 OpenAI-compatible 主链路稳定后追加
- MCP Server / Client
- RAG、Embedding、Vector Store
- 自动执行 Dangerous Command
- 多用户、远程集中式会话与团队权限
- PVE / Kubernetes / Database 新插件

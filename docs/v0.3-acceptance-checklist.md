## 1. 交付摘要

v0.3 围绕 **Docker Plugin + Docker Dashboard + Workflow 增强** 三条主线交付：

- **Docker Plugin**（新插件）：`plugins/docker`，独立 gRPC 子进程
  - 第一阶段：`docker.list` / `docker.inspect` / `docker.start` / `docker.stop` / `docker.restart` / `docker.logs`（流式）
  - 第三阶段：`docker.rm`（Dangerous · 双重护栏）/ `docker.pull` / `docker.push`（流式 progress · X-Registry-Auth）/ `docker.exec`（双向流 · TTY / mux / resize / exit_code）
  - 只读补齐：`docker.images` / `docker.volumes` / `docker.networks`
- **Docker Dashboard**（Desktop 新增 Tab）
  - 第二阶段：容器列表 → Inspect 抽屉 → 流式 logs 面板 → 生命周期弹窗二次确认
  - 第三阶段：Remove 弹窗（force / volumes 可选，Dangerous 双重护栏）+ Exec 交互式终端（xterm.js + winch）+ Images / Volumes / Networks 只读 Tab
- **Workflow 引擎增强**（分五批合入）
  - `when: <expr>` 条件分支（第一批）
  - `retry: { max, backoff, max_backoff, exponential }`（第二批）
  - 执行历史持久化（JSONL，Store 抽象保留 SQLite 切换空间）（第三批）
  - `on_failure` / `rollback` 声明式回滚（第四批）
  - `parallel: true` 组内并行（第五批）
- **配套变更**
  - `core/connection`：新增 `DockerCredentials`（host / api_version / TLS）
  - `apps/cli`：`mow workflow history list|show`
  - `apps/desktop`：Workflow History 面板 + Docker 全 Tab UI
  - `core/command`：根据 `CommandSpec.InputSchema` 执行 JSON Schema 输入校验，稳定错误码
  - CI：三个 module（core / apps / plugins）逐个 build/vet/test，Desktop 前端 `npm ci && npm run build`

## 4. 手动验收（分组）

### 4.1 Docker Plugin（CLI）

- [x] `mow target add dk-local --type docker --docker-host unix:///var/run/docker.sock` 保存成功
- [x] `mow run docker.list --target dk-local --param all=true` 返回 JSON 数组
- [x] `mow run docker.inspect --target dk-local --param id=<id>` 原样透传 Engine JSON
- [x] `mow run docker.start|stop|restart --target dk-local --param id=<id>` 走通；304 → `already_in_state=true`
- [x] `docker.logs` follow 模式收到 mux 帧；`docker.logs tty=true` 原样透传
- [x] `docker.rm` 未带 `confirmed=true` → `CONFIRMATION_REQUIRED`
- [x] `docker.pull` 无 auth 时头字段 `X-Registry-Auth=base64("{}")` 存在；`unauthorized` → `DOCKER_UNAUTHORIZED`
- [x] `docker.exec tty=true` 打印 `top -b -n 1` 输出，退出码回填 `exec_result.ExitCode`

### 4.2 Docker Dashboard（Desktop）

- [x] Docker Tab 只在 active target 类型为 `docker` 时可点
- [x] 容器列表默认 `all=true`，state 徽标 (running/exited) 分色
- [x] Inspect 抽屉展示只读 JSON；关闭抽屉不阻塞列表刷新
- [x] Logs 面板事件推送 stdout / stderr / exit；切换容器自动 `DockerLogsClose`
- [x] Start / Stop / Restart 弹窗展示 command / container / audit id；未确认不下发
- [x] Remove 弹窗：默认不勾 force，`running/restarting` 自动预勾；`volumes` 可选；`Confirmed=false` 被后端拒绝
- [x] Exec 抽屉：xterm.js 首帧 winch 同步尺寸，关闭抽屉自动 `DockerExecClose`
- [x] Images / Volumes / Networks Tab 只读展示

### 4.3 Workflow 增强

- [x] `when: 'contains(steps.probe.out.stdout, "unhealthy")'` false → `⤼ skipped`，OK=true
- [x] `retry: { max: 3, backoff: 500ms, exponential: true, max_backoff: 5s }` 用尽后步骤失败，`Attempts` 与 `PhaseRetry` 事件次数一致
- [x] `on_failure.rollback: [upload, deploy]` 只对成功过的 step 调用 `compensate`；`Skipped` step 不回滚
- [x] `compensate` 内部错误不嵌套 rollback，记录到 `Result.Rollback` 后继续
- [x] 组内 step 引用兄弟 `steps.<sibling>.out.*` → `LoadBytes` 拒绝
- [x] parallel 组 fail-fast：一个失败后同组其它步骤通过 ctx cancel 立即返回
- [x] `Result.Steps` 按声明顺序追加（非完成顺序）
- [x] `mow workflow history list --limit 5` / `show run-<hex>` / `--json` 正常
- [x] Desktop History 面板 Run 结束后自动刷新，点击行展开详情抽屉

### 4.4 边界 / 错误码

- [x] `docker.rm` 走 Command Engine Dangerous 二次确认 → 未 confirmed → 拒绝
- [x] `docker.pull` errorDetail 行触发插件立即返回 `sdk.Error` 而非 `Finish`
- [x] `parallel + retry` 组合：一个步骤重试用尽 → 组内其它步骤 ctx canceled
- [x] `parallel + rollback` 组合：只对最终成功的兄弟执行 compensate
- [x] **npipe 三重护栏**：
  - `TargetsPage` 输入 `npipe://` → 输入框下方黄字提示 + Save 按钮点击后阻止保存
  - 后端 `DockerExecOpen` 若被绕过前端调用（如 CLI）→ 直接错误 `docker.exec: npipe transport is not supported ...`
  - 插件层运行时若被再绕过 → `sdk.Error{Code: DOCKER_NPIPE_UNSUPPORTED / DOCKER_EXEC_NPIPE_UNSUPPORTED}`
- [x] **TLS exec 双重护栏**：
  - `DockerExecDrawer` 挂载后调 `DescribeDockerTarget`，`exec_supported=false` → Start 按钮禁用 + 展示 `exec_unsupported_reason`
  - 插件层 `docker.exec` 直接返回 `sdk.Error{Code: DOCKER_EXEC_TLS_UNSUPPORTED}`
